# Доки по проекту + критерии

Папка для общей документации проекта: контекст, план работ, глоссарий, критерии оценки и исходные материалы.

## Что здесь хранится

- `project-context.md` - зачем нужен проект и какую задачу решает eval.
- `implementation-plan.md` - план MVP и ближайшие шаги.
- `glossary.md` - термины проекта.
- `criteria-intake-notes.md` - консолидированные заметки по критериям.
- `source-documents-notes.md` - список исходных документов.
- `source-documents/` - оригиналы PDF/DOCX/PNG/TXT.
- `source-documents-md/` - Markdown-копии для быстрого чтения и поиска.

## Промты для сторонних ИИ

- `external-ai-project-onboarding-prompt.md` - стартовый промт для первой передачи проекта другой ИИ. Его задача: быстро объяснить цель проекта, структуру БД, критерии, TOP benchmark и ограничения. Использовать до передачи конкретных файлов на оценку.
- `external-ai-evaluator-prompt.md` - рабочий промт для оценки нового eval-раунда или ревью наших оценок. Использовать после onboarding-промта, когда другая ИИ уже поняла контекст проекта.

Значимые правила, на которые должны опираться сторонние ИИ:

- `scoring-methodology.md` - шкала оценок 0..10 по критериям.
- `reporting-rules.md` - правила отчетов, TOP benchmark и вердикта.
- `new-round-workflow.md` - процесс обработки команды "Новый раунд".
