# Факторы прогона ИИ

Документ фиксирует, какие настройки ИИ мы сохраняем для каждого eval-прогона.

## Что относится к ИИ-прогону

| Фактор | Что фиксируем в БД | Поле |
| --- | --- | --- |
| Формулировка промпта и примеры-образцы | Полный текст промпта на момент прогона, включая few-shot примеры, если они есть | `prompt_snapshot` |
| Модель | Название и версия модели | `model_name`, `model_version` |
| Температура | Степень случайности генерации | `temperature` |
| Top-p | Ограничение вероятностной выборки | `top_p` |
| Режим работы | Прямое извлечение, поиск по базе или гибридный режим | `run_mode` |
| Размер фрагмента | Размер фрагмента для retrieval/RAG | `retrieval_chunk_size` |
| Число извлекаемых фрагментов | Сколько фрагментов поднимаем из базы | `retrieval_top_k` |
| Переранжировщик | Какой reranker использовался, если использовался | `reranker_name` |
| Строгость формы ответа | Насколько жестко агент обязан вернуть структуру с полями связь/вид/направление | `response_format_strictness` |
| Контракт ответа | JSON-описание обязательных полей ответа, если нужна детализация | `response_contract_json` |
| Изменения | Короткое описание, что изменили относительно прошлой версии прогона | `change_summary` |
| Агент | Название и версия агента, который выполнял прогон | `agent_name`, `agent_version` |
| Версия датасета | Snapshot версии датасета на момент прогона | `dataset_version` |

## MVP-поля eval_runs

Обязательные:

- `agent_name`
- `agent_version`
- `prompt_snapshot`
- `model_name`
- `model_version`
- `temperature`
- `top_p`
- `run_mode`
- `response_format_strictness`

Опциональные:

- `retrieval_chunk_size`
- `retrieval_top_k`
- `reranker_name`
- `response_contract_json`
- `change_summary`
- `dataset_version`

## Допустимые значения

`run_mode`:

- `direct_extraction` — прямое извлечение без поиска по базе;
- `rag_search` — поиск по базе/RAG;
- `hybrid` — смешанный режим.

`response_format_strictness`:

- `strict` — жесткий контракт ответа, обязательные поля должны быть заполнены;
- `medium` — структура ожидается, но допускаются частичные пропуски;
- `freeform` — свободный ответ, структура проверяется постфактум.
