## TOP benchmark

TOP benchmark - это лучшая предыдущая версия агента по текущей методике оценки. Она используется как точка сравнения, чтобы новый раунд оценивался не только относительно прошлого запуска, но и относительно лучшего результата, который уже был получен ранее.

| Поле | Значение |
|---|---|
| Режим сравнения | автоматический TOP benchmark |
| TOP benchmark | `v3` |
| Почему выбран | Общая полнота набора: 8.7/10; чистота набора: 6.6/10; достоверность отдельных ТК: 9.2/10 |

Текущий раунд `v4` сравнивается с `v3`, которая сейчас является TOP benchmark.

Начиная с третьего раунда benchmark выбирается среди всех предыдущих раундов: сначала по общей полноте набора, затем по чистоте набора, затем по достоверности отдельных ТК.

## Вердикт

Сравнение текущего раунда `v4` с TOP benchmark `v3`.

### Что улучшилось

- Атомарность шагов: 8.9 -> 9.4 (+0.5)
- Ожидаемый результат: 8.7 -> 8.9 (+0.2)
- Достоверность: 9.2 -> 9.9 (+0.7)

### Что ухудшилось

- Позитивное покрытие: 8.9 -> 7.9 (-1.0)
- Негативное покрытие: 9.0 -> 4.0 (-5.0)
- Чистота набора: 6.6 -> 4.0 (-2.6)
- Покрытие обязательных проверок: 10.0 -> 8.6 (-1.4)
- Общая полнота набора: 8.7 -> 6.4 (-2.3)
- Корректность вида ТК: 9.8 -> 9.3 (-0.5)
- Шаблон и обязательные атрибуты: 6.5 -> 6.0 (-0.5)
- Предусловия и постусловия: 8.2 -> 8.0 (-0.2)

### Без существенных изменений

- Критериев без существенной динамики нет.

### Общий вывод

Изменение агента в текущем раунде привело к следующему результату: общая полнота набора изменилась с 8.7 до 6.4; чистота набора изменилась с 6.6 до 4.0; позитивное покрытие изменилось с 8.9 до 7.9; покрытие обязательных проверок изменилось с 10.0 до 8.6.
В целом текущая версия выглядит хуже benchmark `v3`: снижение общей полноты требует анализа причин.
